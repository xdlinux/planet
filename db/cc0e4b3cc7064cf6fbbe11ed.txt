<blockquote><h2>Chapter 9 - 斜杠命令的参数：Options 和 Choices</h2>
<h3>注册斜杠命令是前提</h3>
<p>Discord 为开发人员提供了创建客户端集成斜杠命令的选项。 在本节中，我们将介绍如何使用 discord.js 注册这些命令！任何斜杠命令在可以被使用前，都需要先向 Discord 服务器注册。命令在没做出改动的情况下，不需要每次启动机器人之前注册一遍命令。</p>
<h3>选项 Options</h3>
<p>应用程序命令可以有选项。你可以将这些选项视为函数的参数并指定它们，如下所示：</p>
<p>比如 <code>commands/echo.js</code>：</p>
<pre><code>const { SlashCommandBuilder } = require('discord.js');

module.exports = {
	data: new SlashCommandBuilder()
		.setName('echo')
		.setDescription('Replies with your input!')
		.addStringOption(option =&gt;
			option.setName('input')
				.setDescription('The input to echo back')
				.setRequired(true)),
	async execute(interaction) {
		const input = interaction.options.getString("input");
		await interaction.reply(input);
	},
};
</code></pre>
<p>我们为这个斜杠命令设置了命令名：<code>echo</code>，设置了简介说明：<code>Replies with your input!</code>。</p>
<p>除此之外，我们为这个命令添加了 option 用于接受命令参数，就像 c 语言函数接收参数那样。</p>
<p>这里只有一个 option，类型是字符串类型，只接收字符串（由 <code>.addStringOption</code> 看出）。我们为这个 option 设置了名称 <code>input</code>，设置了说明 <code>The input to echo back</code>，并设置其为必填（由 <code>.setRequired(true)</code> 指定）。</p>

<p>Option 经常和 Choice 搭配，因此下面我们先介绍 Choice，再给出更多的例子。</p>
<h3>选择 Choices</h3>
<p>“选项” 和 “选择” 我感觉都翻译的不咋样，容易弄混，所以我还是用回英文 Option 和 Choice 吧。</p>
<p>Choice 只能作用于 Option。</p>
<p>Choice 只能是 STRING（字符串） 或 INTEGER（整型）类型的 Option 才可以拥有的。</p>
<p>Option 很自由，用户想输入什么合法的内容就接收什么内容。</p>
<p>Choice 就像 C 语言的 <code>enum</code>，用户只能从一些程序给定的选择里挑一个出来。</p>
<p>如果一个 Option 被 Choice “绑定”了，那么 Option 的输入值，有效值只能是 Choices。</p>
<p>我们通过 <code>.addChoice()</code> 方法来构造 Choice。</p>
<p>给个例子：<code>commands/subOption.js</code>：</p>
<pre><code>const { SlashCommandBuilder } = require('discord.js');

module.exports = {
	data: new SlashCommandBuilder()
		.setName('giftest')
		.setDescription('MultiOption test.')
		.addStringOption(option =&gt;
			option.setName("category")
				.setDescription("The choices category")
				.setRequired(true)
				.addChoices(
					{ name: 'Funny', value: 'gif_funny'},
					{ name: 'Meme', value: 'gif_meme'},
					{ name: 'Movie', value: 'gif_movie'}))
		.addIntegerOption(option =&gt;
			option.setName("amount")
				.setDescription("The amount of gifs you want")
				.setRequired(true)
				.setMinValue(1)
				.setMaxValue(5)),
	async execute(interaction) {
		const category = interaction.options.getString("category");
		await interaction.reply(category);
		for (var i=2; i &lt;= interaction.options.getInteger("amount"); i++) {
			await interaction.followUp(category);
		}
	},
};
</code></pre>

<p>这个 Option 它的 “参数名” 是 <code>category</code>，必填，且有效值只可以是由紧跟其后的 Choices 的一项（Funny、Meme 或 Movie）。</p>
<p>这个 Choices 组有三条项目，每条都可以看成是 “键值对”。</p>
<p>同一个 Option 的 Choices 中，name 字段必须唯一，value字段倒是可以相同。</p>

<p>我们通过方法 <code>interaction.options.getInteger("amount")</code> 来获取 amount 的值，这是个用户输入的整数。</p>

<p>我们发现，discord.js 提供的方法丰富多样，对于 bot 开发来说，是够用的。</p>
<p>我们在上面的 <code>commands/subOption.js</code> 中看到，我们通过 <code>.setName("giftest")</code> 设置了 name 字段的值，通过 <code>.setDescription("MultiOption test.")</code> 设置了 description 字段的值，通过 <code>.setMinValue(1)</code> 设置了 minValue 的值。这些都是利用 discord.js 定义好的方法来设置属性的值，它们有个共同特点，就是都是 “set” 开头的方法名称，我们叫它们 set 方法。</p>
<p>类似地，我们通过 <code>.getString("category")</code>、<code>.getInteger("amount")</code> 来获取相应字段的值，它们有个共同特点，就是都是 “get” 开头的方法名称，我们叫它们 get 方法。</p>
<p>set和get一般一起出现，如果只定义了一个会有特殊意义：</p>
<ul>
<li>
<p>如果只有get，表示该属性只可读，不可写</p>
</li>
<li>
<p>如果只有set，表示该属性只可写，不可读</p>
</li>
</ul>

<p>介绍了这么多个类，用代码举个例子 <code>commands/multiOption.js</code>：</p>
<pre><code>const { SlashCommandBuilder } = require('discord.js');

module.exports = {
	data: new SlashCommandBuilder()
	.setName('multioption')
	.setDescription('Multiple options.')
	.addStringOption(option =&gt; option.setName('input').setDescription('Enter a string'))
	.addIntegerOption(option =&gt; option.setName('int').setDescription('Enter an integer'))
	.addBooleanOption(option =&gt; option.setName('choice').setDescription('Select a boolean'))
	.addUserOption(option =&gt; option.setName('target').setDescription('Select a user'))
	.addChannelOption(option =&gt; option.setName('destination').setDescription('Select a channel'))
	.addRoleOption(option =&gt; option.setName('muted').setDescription('Select a role'))
	.addMentionableOption(option =&gt; option.setName('mentionable').setDescription('Mention something'))
	.addNumberOption(option =&gt; option.setName('num').setDescription('Enter a number'))
	.addAttachmentOption(option =&gt; option.setName('attachment').setDescription('Attach something')),

	async execute(interaction) {
		const string = interaction.options.getString('input');
		const integer = interaction.options.getInteger('int');
		const boolean = interaction.options.getBoolean('choice');
		const user = interaction.options.getUser('target');
		const member = interaction.options.getMember('target');
		const channel = interaction.options.getChannel('destination');
		const role = interaction.options.getRole('muted');
		const mentionable = interaction.options.getMentionable('mentionable');
		const number = interaction.options.getNumber('num');
		const attachment = interaction.options.getAttachment('attachment');

		await console.log(string, integer, boolean, user, member, channel, role, mentionable, number, attachment)
		await interaction.reply("done");
	},
};

</code></pre>

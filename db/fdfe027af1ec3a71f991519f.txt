<blockquote><h2>前言</h2>
<p>利用 discord.js 建立一个 Discord 机器人。</p>

<h2>Chapter 1 - 环境搭建</h2>
<h3>环境</h3>
<p>你需要自行下载安装 node.js。</p>
<p>包管理器 npm，yarn，pnpm 有一个就行了。本文使用 npm。</p>

<ul>

<li><a href="https://nodejs.org/">Node.js 16.16.0 LTS</a> ，版本需要至少 16.9.0</li>
<li>包管理器：npm 8.11.0</li>

<p>操作系统可以选择 Linux 发行版、FreeBSD 或 Mac OS，Windows 不是必须的选择，但操作系统你得有。</p>
<p>IDE 也不是必须的，用 VS Code, Sublime Text, Vim, nano 或者记事本都可以。</p>
<p>Node.js 是必须安装的。</p>
<h4>本文编写时，我的云端环境</h4>
<p>云端环境是用来在我的笔记本关机时，让机器人保持继续运行的环境。</p>
<p>云端环境不是必须的，如果你有的话，要保证它能连上 Discord 服务器（可以通过 ping www.discord.com 和 cdn.discordapp.com 来初步验证一下连通性）。</p>
<h3>从 Discord Developer Portal 注册机器人账号</h3>
<p>打开链接 https://discord.com/developers/applications ，登录你的 Discord 账号，选择 <code>New Application</code> 来创建新应用程序。</p>
<p>在 <code>NAME</code> 一栏填写你的应用名称，这个名称将会是机器人的外显账号名称，后期可修改。</p>
<p>按下 <code>Create</code> 以完成创建。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223049452-1249073988.png" alt="image" /></p>
<p>点开新创建的应用，点击左侧菜单栏的 <code>General Information</code>，记下 <code>APPLICATION ID</code>，这就是通常所说的机器人的 "client_id"，这个 ID 是公开可查的，不是秘密。</p>
<p>点击左侧菜单栏的 <code>Bot</code>，点击 <code>Add Bot</code> ，再点击弹出对话框的 <code>Yes, do it!</code>  创建机器人。注意此操作是不可逆的，也就是说机器人一旦创建，就不可被销毁删除。哪怕销毁应用程序，机器人账号仍会在 Discord 的世界里存在，只是变成无人能管理的孤儿账号了。</p>
<p>关于机器人账号和普通用户账号的区别，这里 (https://discord.com/developers/docs/topics/oauth2#bots) 有介绍。</p>
<p>完成机器人账号创建后，在本页面的 <code>Build-A-Bot</code> 栏目中，你可以给机器人账号更换用户名和头像。</p>
<p>点击按钮 <code>Reset Token</code> 并点击 <code>Yes, do it!</code> 来首次获取我们的机器人账号 Token。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202207/2455224-20220711125633911-1411679088.png" alt="image" /></p>
<p><strong>注意：Token (令牌)万万不可给别人看到，这是十分机密的一串文字，任何人有了它，就拥有了完全掌控机器人行为的能力了，比如机器人恶意封禁和私聊服务器里的人、发送 @everyone 等恶劣行为都可能会发生。请妥善保管好这个密钥，</strong></p>
<p><strong>注意：Discord 出于安全目的，令牌在创建时只能查看一次。 如果您忘记或无法访问您的令牌，请重新生成一个新令牌，旧令牌将自动失效。</strong></p>
<p>接着，在 <code>Bot</code> 页面的 <code>Privileged Gateway Intents</code> 栏目中，勾选 <code>Presence Intent</code>、<code>Server Members Intent</code> 和 <code>Message Content Intent</code>。</p>
<p>然后点击下端弹出的绿色保存设置 (Save Changes) 按钮，来保存设置。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223236956-513821563.png" alt="image" /></p>
<p>点击左侧菜单栏的  <code>OAuth2</code> - <code>URL Generator</code> 。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223317282-341711629.png" alt="image" /></p>
<p>在 <code>SCOPES</code> 里勾选 <code>bot</code> 和 <code>applications.commands</code> ，为了方便，在 <code>BOT PERMISSIONS</code> 里直接勾选第一条 <code>Administrator</code> （注意咱们只是为了玩哈，才直接赋予管理员权限的）。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223338452-1398283213.png" alt="image" /></p>
<p>复制最下方生成的 URL，粘贴到新浏览器窗口打开，选择服务器并添加。如果你没有看到任何可选选项，那么你需要去 Discord 先自己创建一个服务器，然后回来刷新一下。事实上，其他服务器如果给你管理服务器的权限的话，这个服务器将出现在可选选项中。</p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223358375-25536375.png" alt="image" /></p>
<p><img src="https://img2022.cnblogs.com/blog/2455224/202210/2455224-20221005223409625-2120951239.png" alt="image" /></p>
<p>到此，我们完成了搭建开发环境、创建机器人账号和邀请机器人进入服务器等操作。</p>
<h2>Chapter 2 - 工程文件夹的初始化</h2>
<h3>创建工程文件夹</h3>
<p>对于 IDE 用户，创建一个 Node.js 项目。对于编辑器用户，新建文件夹即可。</p>
<p>比如我的工程根文件夹是 <code>C:\Users\root\WebstormProjects\discord_bot_2022</code></p>
<h3>获取 discord.js</h3>
<p>本文编写时，discord.js 的最新版本是 v14，<strong>咱们使用的版本也是 v14</strong>。后文的部分代码可能在过去和未来的版本（比如 v13 和 v15）中失效。</p>
<p>不过幸运的是，从 v13 升级到 v14 的情况来看，没啥大问题。何况 v14 才于 2022 年 7 月 18 日正式发布(本文于 2022 年 7 月 9 日开始撰写第一版，7 月 19 日完成针对 v14 的第二版)，v15 的发布还远着呢。</p>
<p><strong>注意：你的 Node.js 的版本需高于 16.9.0（含）</strong></p>
<p>cd 到 <code>工程根文件夹</code>，在终端输入来获取 discord.js：</p>
<ul>
<li>如果你用的是 npm：</li>
</ul>
<pre><code>npm install discord.js @discordjs/rest discord-api-types
</code></pre>
<ul>

</ul>

<h3>打开 Discord 开发者模式</h3>
<p>为了方便地获取服务器 id、身份组 id 、消息 id 和用户 id，我们必须这么做。</p>
<p>网页版在Discord <code>设置</code> - <code>高级设置</code> 里，勾选 <code>开发者模式</code>。</p>
<p>手机版在 <code>DiscordAPP</code> - <code>"我"的头像</code> - <code>APP设置栏</code> - <code>外观</code> - <code>高级设置栏</code> - <code>开发者模式</code>。</p>
<h3>创建配置文件 config.json</h3>
<p>在工程根文件夹下创建文件 <code>config.json</code>，写入：</p>
<pre><code>{  
   "token": "bot-token-goes-here",  
   "clientId": "bot-clientid-goes here",  
   "guildId": "serverid-goes-here"  
}
</code></pre>
<p>比如（这三条都是我瞎编的，不是真的）</p>
<pre><code>{  
   "token": "OBEzNDMyTEN0OTkzOTk1ODA4.Bhz0k7.97sjsWOGtmx-LBIlFvrkk6A9lkk9YGMX9OYbf",  
   "clientId": "240832114973084058",  
   "guildId": "1978274108637234708"  
}
</code></pre>
<p>需要说明的是，<code>token</code> 字段就是那个极为重要不能外泄的机器人密钥，把你的粘贴过来。</p>
<p><code>clientId</code> 字段就是 <code>Application id</code>。在 Discord 服务器里。右击机器人头像，点击 <code>复制 ID</code> 即可轻松获取。</p>
<p><code>guildId</code> 字段是服务器的 ID，当然得是机器人所在的服务器的 ID。右击服务器头像，点击 <code>复制 ID</code> 即可轻松获取。</p>
<h3>.gitignore 的配置</h3>
<p>接下来确保你的工程根文件夹下存在 <code>.gitignore</code> 文件，确保文件内已写入</p>
<pre><code>node_modules
config.json
.env
</code></pre>
<p>第一行的 node_module 防止你把包也给上传了，白白占用空间。</p>
<p>第二行的 config.json 是防止你的 token 在 git 推送后外泄。</p>
<p>第三行有就有吧。</p>
<h3>无脑实现一个机器人</h3>
<p>在 <code>工程根文件夹</code> 创建文件 <code>deploy_commands.js</code>，写入：</p>
<pre><code>const fs = require("fs");  const { REST } = require('@discordjs/rest');  const { Routes } = require('discord.js');  
const { clientId, guildId, token } = require('./config.json');  
  
const commands = [];  //const commandFiles = fs.readdirSync("./commands").filter(file =&gt; file.endsWith(".js"));  
const cmdPaths = require("./cmdPaths.js").data;  
const commandFiles = [];  
for (let i = 0; i &lt; cmdPaths.length; i++) {  
   commandFiles[i] = fs.readdirSync(cmdPaths[i]).filter(file =&gt; file.endsWith(".js")); // fs.readdirSync() 的结果是个数组，所以 commandFiles是个二维数组  
   for (let j = 0; j &lt; commandFiles[i].length; j++) {  
      commandFiles[i][j] = cmdPaths[i] + "/" + commandFiles[i][j];  
   }  
}  
  
//console.log(commandFiles);  
for (const fileArray of commandFiles) {  
   for (const file of fileArray) {  
      console.log(file);  
      let command = require(`./${file}`);  
      commands.push(command.data.toJSON());  
  
      // if any ‘aka' name exists  
      if (command.akaNames != null &amp;&amp; command.akaNames !== []) {  
         for (let i = 0; i &lt; command.akaNames.length; i++) {  
            let akaData = command.data;  
            akaData.name = command.akaNames[i];  
            commands.push(akaData.toJSON());  
         }  
      }  
   }  
}    
    
const rest = new REST({ version: '10' }).setToken(token);  
  rest.put(Routes.applicationGuildCommands(clientId, guildId), { body: commands })    
   .then(() =&gt; console.log('Successfully registered application commands.'))    
   .catch(console.error);
</code></pre>
<p>在 <code>工程根文件夹</code> 下创建文件 <code>cmdPaths.js</code>，写入：</p>
<pre><code>module.exports = {  
    data: ["./commands"],  
};
</code></pre>
<p>在 <code>工程根文件夹</code> 下创建文件夹 <code>events</code>，在这个文件夹里创建文件 <code>ready.js</code>，写入：</p>
<pre><code>module.exports = {  
   name: "ready",  
   once: true,  
   execute(client) {  
      console.log(`Ready! Logged in as ${client.user.tag}`);  
   },  
};
</code></pre>
<p>在 <code>工程根文件夹/event</code> 文件夹里，创建文件 <code>interactionCreate.js</code>，写入：</p>
<pre><code>module.exports = {  
   name: "interactionCreate",  
   execute(interaction) {  
      console.log(`${interaction.user.tag} in #${interaction.channel.name} triggered an interaction.`);  
   },  
};
</code></pre>
<p>在 <code>工程根文件夹</code> 创建文件 <code>index.js</code> ，写入：</p>
<pre><code>const fs = require('fs');  
const { Client, Collection, GatewayIntentBits } = require('discord.js');  
const { token } = require('./config.json');  
const { InteractionType } = require("discord-api-types/v10");  
  
const client = new Client({ intents: [GatewayIntentBits.Guilds, GatewayIntentBits.GuildMessages, GatewayIntentBits.MessageContent] });  
  const eventFiles = fs.readdirSync('./events').filter(file =&gt; file.endsWith('.js'));    
    
for (const file of eventFiles) {  
    const event = require(`./events/${file}`);  
    if (event.once) {  
        client.once(event.name, (...args) =&gt; event.execute(...args));  
    } else {  
        client.on(event.name, (...args) =&gt; event.execute(...args));  
    }  
}  
  
client.commands = new Collection();  
const cmdPaths = require("./cmdPaths.js").data;  
const commandFiles = [];  
for (let i = 0; i &lt; cmdPaths.length; i++) {  
    commandFiles[i] = fs.readdirSync(cmdPaths[i]).filter(file =&gt; file.endsWith(".js")); // fs.readdirSync() 的结果是个数组，所以 commandFiles是个二维数组  
    for (let j = 0; j &lt; commandFiles[i].length; j++) {  
        commandFiles[i][j] = cmdPaths[i] + "/" + commandFiles[i][j];  
    }  
}  
  
for (const fileArray of commandFiles) {  
    for (const file of fileArray) {  
        const command = require(`./${file}`);  
        client.commands.set(command.data.name, command);  
  
        // if any ‘aka' name exists  
        if (command.akaNames != null &amp;&amp; command.akaNames !== []) {  
            for (let i = 0; i &lt; command.akaNames.length; i++) {  
                client.commands.set(command.akaNames[i], command);  
            }  
        }  
    }  
}  
  
client.once('ready', () =&gt; {});  
client.on('interactionCreate', async interaction =&gt; {  
    if (interaction.type !== InteractionType.ApplicationCommand) return;  
  
    const command = client.commands.get(interaction.commandName);  
  
    if (!command) return;  
  
    try {  
        await command.execute(interaction);  
    } catch (error) {  
        console.error(error);  
        await interaction.reply({  
            content: 'There was an error while executing this command!',  
            ephemeral: true  
        });  
    }  
});  
  
client.login(token);
</code></pre>
<p>在 <code>工程根文件夹</code> 下创建文件夹 <code>commands</code>，在这个文件夹里创建文件 <code>ping.js</code>，写入：</p>
<pre><code>const { SlashCommandBuilder } = require('discord.js');  
  
module.exports = {  
   data: new SlashCommandBuilder()  
       .setName('ping')  
       .setDescription('Replies with Pong!'),  
   async execute(interaction) {  
      await interaction.reply("Pong!");  
   },  
};
</code></pre>

<pre><code>node index.js

<p>如果出现 “ 无法将“node”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。请检查名称的拼写，如果包括路径，请确保路径正确，然后再试一次。” 或者 “ 'node' 不是内部或外部命令，也不是可运行的程序或批处理文件。”亦或者“ -bash: node: command not found ”，请检查你的 PATH 是否有包含 node.exe 所在文件夹，这通常是 <code>C:\Program Files\nodejs\node.exe</code>。验证方法是 <code>node -v</code> 输出版本号。</p>
<p>如果抛出其他失败，请检查你的网络是否能连接到 Discord、你的上述步骤有没有漏做或者顺序错误的。</p>
<h3>与机器人交互</h3>
<p>cd 到工程根文件夹。</p>
<p>确保你已经通过 <code>node deploy_commands.js</code> 注册了斜杠命令，并已通过 <code>node index.js</code> （或者 <code>node .</code>） 使机器人程序保持运行状态。验证方法是：能在 Discord 服务器成员列表看到你的机器人是在线状态而非离线状态。</p>
<p>打开 Discord，在机器人所在的服务器的任何文字频道，输入 <code>/ping</code>，回车选定斜杠命令，再次回车发送命令，机器人应该会发送消息 <code>Pong!</code> ，而且在机器人后端程序，会输出类似于 <code>xxxxx#xxxx in #bot triggered an interaction.</code></p>
